# 港股 + 美股 对称信号策略说明（日 K + 60m，纯技术信号）

本策略为**日频、纯技术面、多头**，**不含基本面**，且**当前版本刻意不含**固定止损、移动止盈、盈亏相关离场——**仅由日 K 与 60m 的对称技术条件**决定开平仓，便于检验「信号本身」是否有效；风控可在验证有效后再叠加。**买卖成交价**为交易日**第二根 60m K 的开盘价**（Longport `Min_60`）。下文为默认参数下的逻辑与计算方法。

---

## 一、策略概要

在每个交易日，在允许**新开仓**的前提下，从**候选池**中选出满足**日 K 突破 + 趋势 + 量能 + 流动性**，并叠加**60m** 上与入场**镜像**的条件（短周期均线、小时级上下破、可选首根阴阳）建仓。**平仓**则当日 K 出现**同窗口通道下破**或**趋势空头**，或 60m 上出现**与入场对称的空头信号**（任一侧触发即可平仓）。仓位可随**波动率目标**缩放（默认关闭缩放，见第五节）。

---

## 二、选股与候选池

1. **推荐默认（恒指 ∪ 恒生科技）**  
   - **恒指成分**：以恒生指数公司公布的成分名单为准（需自行维护成分表 CSV）。  
   - **恒生科技成分**：同上。  
   - 两者取**并集**去重；恒指与恒生科技重叠较多，**合并后约 90± 只量级**。

2. **自选列表**：仅交易你指定的代码列表（`UNIVERSE_MODE=csv` 与股票池 CSV）。

**每日可交易池**：在候选池内，再按**截至昨收**的**价格区间**与**近若干日平均成交额区间**（港股、美股分别设定上下界）过滤；只有同时满足流动性条件的标的才会进入当日买入池。

---

## 三、时间与信号计算（避免未来函数）

### 日 K（入场与出场对称）

- 回测日 \(T\) 上，日频指标仅使用**不晚于 \(T-1\)** 的日线。  
- **入场**：**突破**（收盘高于前一日滚动 \(N\) 日最高价，与旧版一致）、**趋势多头**（收盘高于 \(P\) 日均线）、**量比** ≥ 阈值、**流动性**过滤。通道宽度为 **`breakout_lookback`（\(N\)）**。  
- **出场（信号）**：与上对称 —— **下破 \(N\) 日最低价通道**（`is_breakdown`，与突破同构），或 **趋势空头**（收盘低于同一根趋势均线）。**不设**固定止损比例与盈亏条件。

### 60m K（与入场镜像的出场 + 成交价）

- **早于 \(T\) 的已收盘 60m 柱**：若启用 `hourly_ma_period`，则入场要求末根收盘在短均线上方，出场要求末根收盘在短均线**下方**；若启用 `hourly_breakout_bars`，则入场为小时级上破、出场为**同构下破**。  
- **当日首根 60m**：若启用 `use_hourly_first_bar_bullish`，入场需首根**收阳**，出场镜像为首根**收阴**（第二根开盘前已知）。  
- **成交价**：信号成立日 \(T\) 以**第二根 60m 开盘价**成交；数据不足时回退 **T−1** 日收。

**平仓逻辑**：满足**日 K 出场**或**60m 出场**（与各自入场开关对应）**之一**即卖出；评估顺序上日 K 条件优先用于归因展示。

训练脚本默认网格联合搜索 **通道宽、量比、`hourly_ma_period`** 等；`--exp-hourly` 扫描更细的 60m 组合。

---

## 四、大盘环境过滤（默认关闭）

若开启 `USE_REGIME_FILTER`，则仅在 **恒生指数** 与 **标普 500 ETF（SPY）** 等基准满足规则时允许**新开仓**；持仓仍可按**信号**出场。

---

## 五、波动率目标仓位（默认关闭）

默认 **`VOL_TARGET_ANNUAL = 0`**，不做波动缩放，便于纯信号检验。若设为大于 0，则按实现波动缩放单笔名义仓位（高波动少买、低波动略多买）。

---

## 六、出场规则（纯信号，与入场对称）

仅当 **日 K** 或 **60m** 出现上节所述的**空头/下破信号**时平仓，**无**止损比例、移动止盈、MA60 亏损离场等。

---

## 七、仓位与交易单位

- 最多同时持有**最多持仓数**只；单笔目标为总权益的**仓位比例**，再乘以波动率缩放。  
- 港股按**整手 100 股**近似；美股按**整股**。  
- 单边费率（默认 0）可按买卖各一次自行理解口径。

---

## 八、业绩基准（与策略对比用）

报告中「基准」为 **恒指与美股 SPY 收盘价等权、各自归一化到起点 100 后再取平均**得到的组合曲线，用于粗略对比跨市场买入持有风格。

---

## 十、局限与免责声明

- 历史回测不预示未来；股票池过小或时段过短会**严重扭曲**夏普、回撤与年度胜率。  
- 若首段数据始于某年中，该「首年」可能是**非完整自然年**。  
- 当前默认参数为**风险与收益权衡**下的一组设定，**不保证**在任何样本外区间或实盘中仍为最优。  
- **样本划分（可选做法）**：训练窗口在 `train_params.py`；主回测区间只在 `backtest.py` 配置，二者分离。
- **日线「更早」不是混淆**：回测/训练会按 `DATA_WARMUP_DAYS_BEFORE_START`（默认 400 自然日）**早于**交易起点拉取日线，用于滚动指标预热；**策略交易与净值仍只从 `BACKTEST_START`（默认 2024-04-01）起统计**。**60m** 请求起点为 `max(日线预热起点, TREND_HOURLY_MIN_DATE)`，与小时数据可用区间一致；日志中会分别打印「日线请求」「锚点」「60m 请求」。
- **Longport 60m**：分钟/小时 K 存在**历史起点**（常见错误码 301600），代码中默认以环境变量 **`TREND_HOURLY_MIN_DATE`**（默认 **`2024-04-01`**）抬高请求起点；若 API 仍报错，可再改晚。单次接口约**千根**上限，`_fetch_hourly_range` 已按 **`LONGPORT_HOURLY_BATCH_DAYS`**（默认 25）分批拉取并合并。若希望「完全重拉」某段缓存，可手动删除 `data_cache/daily/` 或 `data_cache/hourly_60/` 下对应文件（见 README 数据目录说明）。  
- **`relax_hourly_when_incomplete`**（默认开）：当某日 60m 根数不足或 T 前历史不足以算均线/突破时，不强行拦截（便于与缺数据区间共存）；关闭后更严格。  
- **Sharpe 与总收益率**：报告中的年化 Sharpe(Rf=0) 由**逐日净值变动**算日简单收益，再 `√252 × 均值/标准差`，**不是**用「总收益 ÷ 天数」直接换算。若大量交易日**几乎现金、净值几乎不变**，日收益方差会很小，**均值/标准差**仍可能较大，Sharpe **看起来不低**，但总收益可能只有几个点——此时 Sharpe **不宜单独解读**，应结合交易笔数、回撤与区间长度一起看。
